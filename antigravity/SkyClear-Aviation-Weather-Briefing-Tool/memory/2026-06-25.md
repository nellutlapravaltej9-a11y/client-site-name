# 2026-06-25 - SkyBrief Aviation Weather Briefing Tool Bug Fixes

## What Happened
- Investigated and fixed multiple parsing and UI rendering issues in the SkyBrief aviation weather decoder application.
- Successfully verified the application builds and passes all type checks.

## Details of Changes
1. **Pressure Parsing Fix**: Modified altimeter regexes in `metarDecoder.ts` (`/^A\d{4}$/` and `/^Q\d{4}$/`) to correctly capture standard 5-character readings like `A2992` or `Q1013`, resolving a bug where pressure always defaulted to `29.92`/`1013 hPa`.
2. **PROB TEMPO Combining**: Enhanced TAF period parser in `tafDecoder.ts` to merge contiguous `PROB30` or `PROB40` and `TEMPO` tokens (e.g. `PROB30 TEMPO`) so probability data is not discarded.
3. **BASE Group Leak Prevention**: Cleaned up the BASE group tokens by filtering out issue times (`Z` suffix) and validity ranges (`\d{4}/\d{4}`), preventing header leakage into METAR decoding.
4. **Recharts Sizing Fix**: Added `esbuild: { keepNames: true }` to `vite.config.ts` to prevent minification of chart component displayNames in the production bundle. This fixes the issue where charts loaded as blank/empty blocks inside `<ResponsiveContainer>` on Vercel.
5. **ICAO Scanning Robustness**: Updated `decodeMETAR` to scan the first 4 tokens for the first 4-letter alphabetical word (excluding standard METAR metadata keywords), avoiding errors when reports start with correction markers like `COR`.
6. **Theme Colors**: Standardized the category sleek theme in `BriefingView.tsx` to match aviation rules (IFR is Red `#EF4444`, LIFR is Purple `#A855F7`).
7. **Local Generated Time**: Fixed `{timestamp.split(' ')[1] || 'Live'}` to display `{timestamp}` directly in `BriefingView.tsx` since time is formatted as `HH:mm:ss`.
8. **HTML Title**: Updated generic title to `"SkyBrief — Aviation Weather Briefing Tool"`.
